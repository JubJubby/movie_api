Objective

To build the server-side component of a “movies” web application. The web application will provide users with access to information about different movies, directors, and genres. Users will be able to sign up, update their personal information, and create a list of their favorite movies.

The APIs are made using RESTful architechture.

  • /movies returns a list of all movies using a GET request.

  • /movies/movieName returns a movie's data by the movie's name using a GET request.
    Use "%20" in place of a space.
    Example: "/movies/Rat%20Race" will return:
    Title: "Rat Race"
    Director: "Jerry Zucker"
    Genre: "Comedy"

  • /movies/genre/genreName returns movies by the movies' genre using a GET request.
    Use "%20" in place of a space.
    Example: "/movies/genre/Horror" will return:
    Title: "Donnie Darko"
    Director: "Richard Kelly"
    Genre: "Horror"

  • /movies/directors/directorName returns director's data by name using a GET request.
    Use "%20" in place of a space.
    Example: "/movies/directors/Mel%20Brooks" will return:
    Title: "Spaceballs"
    Director: "Mel Brooks"
    Genre: "Comedy"

  • /users creates a new JSON object holding data about the user including an ID using a POST request.
    Example:
    ID: 1
    Username: John Doe
    Password: Password123!
    email: jondoe@email.com
    Birthday: 01-02-03
    Favorite Movies: []

  • /users/ID updates a user's information using a PUT request.
    Example:
    ID: 1
    Username: Jon Doe
    Password123!
    email: jondoe@email.com
    Birthday: 01-02-03
    Favorite Movies: []

  • /users/ID/movieName adds a movie to a user's favorites list using a POST request. This returns only a text saying that the movie has been added.

  • /users/ID/movieName removes a movie from a user's favorites list using a DELETE request. This returns only a text saying that the movie has been removed.

  • /users/ID removes a user using a DELETE request. This returns only a text saying that the user has been deleted.
